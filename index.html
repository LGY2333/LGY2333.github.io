如何创建个人博客（github page + hexo）

该篇文章将介绍如何使用github作为服务器去托管我们的本地静态博客，并绑定域名。其中hexo是博客框架。

优点：上手简单

缺点： github访问不稳定

1. 配置环境

1.1 下载 git 和 nodejs, 直接官网下载就行，然后验证是否下载成功

$ node -v
v18.12.1

$ git -v
git version 2.38.1.windows.1

1.2 生成SSH key，一路Enter，复制默认位置的public key(后缀为pub）到github上（右上角setting -> SSH -> NEW SSH key）

$ ssh-keygen -t rsa -C "1887148xxxx@163.com" 邮箱地址为github注册邮箱
默认生成位置C:\Users\用户名\.ssh\id_rsa

$ ssh -T git@github.com 验证是否添加成功，问题直接yes
Hi LGY2333! You've successfully authenticated

2. 本地配置博客

2.1 使用npm(nodejs自带)下载hexo，然后验证是否下载成功

$ npm install hexo-cli -g
$ hexo -v
hexo: 6.3.0

2.2 创建一个文件夹用于放自己的blog，该路径下执行命令以初始化hexo博客框架。此时文件夹下面应该会生成以下文件

$ hexo init
INFO  Start blogging with Hexo!

source:源文件此处为默认内容（hello world）
public:hexo生成出来的文件
_config:个人配置文件

2.3 执行命令配置本地博客，博客内容为默认。通过这个网址就可以看到本地静态博客的内容了

$ hexo s
 Hexo is running at http://localhost:4000/

3. 自动部署博客到github上，并绑定域名

3.1 github上创建用于存储静态博客的仓库，名字要注意格式，选上Add a README，否则为空仓库

repository: LGY2333.github.io LGY2333换成自己的用户名称

3.2 修改本地个人配置文件_config，将默认deploy语句换成以下部分，repository换成自己的

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: git
  repository: https://github.com/LGY2333/LGY2333.github.io.git
  branch: main

3.3 下载自动部署工具

$ npm install hexo-deployer-git --save

3.4 生成静态博客内容，并将其上传至github。之后就可以通过固定网址访问(网址：用户名.github.io)

$ hexo g
生成静态博客内容
可能会报错 Author identity unknown，执行命令
git config --global user.email "github绑定的邮箱"
git config --global user.name "github用户名"

$ hexo d
将其上传到github上
可能会报错 OpenSSL SSL_read: Connection was reset, errno 10054
这种错误多半是因为访问github超时，多尝试几次一般就可以成功，这也是这种方式的劣势（连接不稳定）
或者直接执行下面的命令
$ git config --global http.sslVerify "false"
直到出现 INFO  Deploy done: git 代表成功

4. 域名绑定

4.1 申请自己的域名

阿里云或者腾讯云都可以，此处略过

4.2 github仓库中的静态博客绑定私人域名

本地文件夹source中，添加文件CNAME,注意文件类型没有后缀

之后使用命令将文件生成在public中，随后上传到github中。

$ hexo g
$ hexo d

这里没有使用网上较多的仓库直接添加CNAME文件的方式，因为每次hexo d之后仓库更新，该文件就需要重新添加一次;

4.3 获取之前固定网址的ip

$ ping 用户名.github.io
Reply from xxx.xxx.xxx.153

4.4 把私人域名解析的地址改为上述ip，之后就可以通过私人域名访问静态博客了

具体位置为阿里云-> 域名列表 -> 选中当前域名 -> 解析 -> 添加记录，一般是两条，@和www

5. 更新静态博客内容

需要注意要先把之前生成的静态博客内容删除，即删除public文件

$ hexo cl 删除
$ hexo g
$ hexo d
